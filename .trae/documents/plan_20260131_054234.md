# Flutter本地推理重构方案（Ollama + MiniCPM4-0.5B）

## 技术选型
**框架**：Ollama（成熟稳定，提供REST API，无需使用Ollama App）
**模型**：MiniCPM4-0.5B（2025年6月最新，性能最强，内存占用最小）

## 实施步骤

### 1. 移除现有推理代码
- 删除llama_cpp_new_client.dart及相关代码
- 清理pubspec.yaml中的旧依赖

### 2. 集成Ollama
- 在iOS和Android端分别安装Ollama CLI/Server
- 配置Ollama服务（本地运行，默认端口11434）
- Flutter通过HTTP API调用Ollama（无需Ollama App）

### 3. 下载模型
- 从ModelScope下载MiniCPM4-0.5B模型：`modelscope download --model OpenBMB/MiniCPM4-0.5B --local_dir ./models`
- 或直接使用Ollama加载：`ollama run openbmb/minicpm4-0.5b-qat-int4-gptq-format`
- 验证模型加载成功

### 4. 实现Flutter客户端
- 创建新的LlmClient使用Ollama REST API
- 实现模型加载、推理接口
- 添加流式输出支持
- 实现错误处理和重试机制

### 5. 优化和测试
- 实现模型缓存机制
- 添加内存监控和警告
- 在iOS设备上测试推理性能
- 测试不同量化级别（Q4、Q3等）

## 模型优势
- 性能最强：超Qwen3-0.6B
- 内存最小：约400-500MB
- 速度最快：220倍提速
- 最新发布：2025年6月
