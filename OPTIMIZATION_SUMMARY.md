## 2. 用户界面设置面板 - QA内容范围选择

### 功能说明
在AI设置面板中新增"问答内容范围"配置，允许用户自定义AI问答时参考的阅读内容范围。

### 界面位置
`AI伴读` → `AI设置` → `问答内容范围`

### 三个选项

#### 📄 选项1：仅当前页面
- **名称：** 仅当前页面
- **内容：** 只包含用户当前看到的页面内容
- **适用场景：** 回答具体细节问题
- **精度：** 高（最精确）
- **上下文：** 少

#### 📖 选项2：当前章节到当前页
- **名称：** 当前章节到当前页
- **内容：** 从章节开头到当前页面的所有内容
- **适用场景：** 需要理解章节上下文的问题
- **精度：** 中
- **上下文：** 中

#### 🎯 选项3：前5页+当前页+后5页（默认）
- **名称：** 前5页+当前页+后5页
- **内容：** 当前页面前后各5页（共11页）滑动窗口
- **适用场景：** 平衡精确性和上下文，适合大多数问答
- **精度：** 中低
- **上下文：** 丰富

### 界面截图描述
```
┌─────────────────────────────────┐
│ 问答内容范围                    │
│                                 │
│  ○ 仅当前页面                   │
│  ○ 当前章节到当前页             │
│  ● 前5页+当前页+后5页           │
└─────────────────────────────────┘
```

### 技术实现

#### 枚举定义
```dart
// lib/presentation/providers/ai_model_provider.dart
enum QAContentScope {
  currentPage,              // 仅当前页面
  currentChapterToPage,     // 当前章节到当前页
  slidingWindow,            // 滑动窗口（前5页+当前+后5页）
}
```

#### 设置管理
- 使用 `SharedPreferences` 自动保存用户选择
- Key: `_kQAContentScope`
- 默认值: `slidingWindow`

#### UI组件
- `_qaContentScopeSettings()` - 设置面板容器
- `_scopeChip()` - 单选按钮组件
- 使用 `Consumer<AiModelProvider>` 实现响应式更新

### 使用方法
```dart
final aiModel = context.read<AiModelProvider>();

// 获取当前设置
final scope = aiModel.qaContentScope; // 返回 QAContentScope

// 设置范围
await aiModel.setQAContentScope(QAContentScope.currentPage);

// 在QA服务中使用
final qaService = QAService(
  contextService: contextService,
  credentials: credentials,
  contentScope: aiModel.qaContentScope, // 自动使用用户设置
);
```

### 优势
- ✅ 用户可自定义AI问答的参考范围
- ✅ 不同场景选择不同范围，提升回答质量
- ✅ 设置自动保存，下次打开应用仍然有效
- ✅ 界面直观，操作简单
- ✅ 默认选项（滑动窗口）适合大多数场景

## 3. XML内容处理机制优化

### 优化内容
全面升级XML内容清理工具，确保只保留有效的阅读文本内容。

### 核心功能

#### 1. 多层清理策略
```dart
String cleanXmlContent(String xmlContent) {
  // 第一层：标准XML清理
  // - 移除XML声明
  // - 移除注释
  // - 移除CDATA
  // - 提取标签间文本
  // - 转换HTML实体

  // 第二层：空内容检测
  // - 如果结果为空或太短，启用激进提取

  // 第三层：有意义文本提取
  // - 逐行检查
  // - 跳过无意义行
  // - 只保留有意义的内容
}
```

#### 2. 智能内容检测
```dart
// 检测是否为有意义的文本
static bool _looksLikeMeaningfulText(String text) {
  // 必须包含字母、中文或数字
  // 特殊字符不能超过30%
  // 长度至少5个字符
}
```

#### 3. 无意义内容过滤
**自动移除：**
- ❌ XML标签（`<tag>`）
- ❌ XML声明（`<?xml...?>`）
- ❌ 注释（`<!-- -->`）
- ❌ CDATA块（`<![CDATA[...]]>`）
- ❌ HTML实体（`&nbsp;`, `&lt;`等）
- ❌ 零宽字符和控制字符
- ❌ 特殊字符组合（`***`, `---`, `___`）
- ❌ 替换字符（`�`）
- ❌ 过短的行（少于5个字符）
- ❌ 仅包含符号的行

#### 4. 空内容优化处理
当标准清理返回空内容时，自动启用激进提取模式：
```dart
// 逐行扫描，只提取看起来有意义的文本
static String _extractMeaningfulText(String content) {
  // 1. 按行分割
  // 2. 清理每行的标签
  // 3. 检查长度和字符组成
  // 4. 只保留有意义的行
  // 5. 合并结果
}
```

### 处理示例

#### 输入（XML）
```xml
<?xml version="1.0"?>
<chapter>
  <!-- 这是一个注释 -->
  <p>第一章 开始</p>
  <p>这是一个故事...</p>
  <div class="empty"></div>
  <script><![CDATA[console.log('test');]]></script>
</chapter>
```

#### 输出（纯文本）
```
第一章 开始 这是一个故事...
```
